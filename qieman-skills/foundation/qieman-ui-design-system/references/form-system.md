# Form System 表单组件体系

## 1. 定义

表单用于收集用户输入、选择条件、确认交易参数。金融表单必须降低误填、误触和误解风险。

## 2. 表单类型

| 类型 | 适用场景 |
|---|---|
| Text Input | 昵称、备注、搜索 |
| Amount Input | 买入金额、回购金额、转出金额 |
| Percent Input | 跌幅、比例、仓位 |
| Selector | 银行卡、账户、策略、期限 |
| Checkbox | 协议、授权、确认阅读 |
| Radio / Segment | 单选配置、快捷金额 |
| Switch | 自动跟车、订阅、提醒开关 |
| Stepper / Group | 增加一组、条件组合 |

## 3. 输入框尺寸

| 场景 | 高度 | 圆角 | Padding |
|---|---:|---:|---:|
| 普通输入 | 44px - 48px | 8px - 12px | 12px - 16px |
| 金额输入 | 56px - 64px | 12px - 16px | 16px |
| 搜索框 | 36px - 40px | 18px - 20px | 12px |
| 底部弹层输入 | 48px - 56px | 12px | 16px |

## 4. Token Mapping

| 属性 | 推荐 Token |
|---|---|
| 输入框背景 | `background/card/default` |
| 输入框边框 | `border/default` |
| 聚焦边框 | `border/primary` |
| 错误边框 | `border/error` |
| 错误说明 | `foreground/error` |
| 占位文字 | `foreground/neutral subtlest` |
| 文本 | `foreground/neutral` |
| 圆角 | `Radius/Medium` / `Radius/Large 1` |

## 5. 金额输入规范

金额输入应优先表达币种与单位：

```text
¥ [金额]
最低买入 100 元
可用余额 1,234.56 元
预估手续费 1.50 元
```

规则：

- 金额输入支持千分位；
- 空值时显示明确占位；
- 输入后实时计算相关费用或触发条件；
- 错误提示出现在输入框下方；
- 涉及余额不足时给出解决路径。

## 6. 百分比输入规范

适用于跌幅买入、回购条件、资产比例等。

- 输入范围必须明确，例如 `0-99%`；
- 与基准净值相关时，应展示计算结果；
- 不得只显示公式，必须给用户可理解的解释。

## 7. 表单校验

| 状态 | 展示方式 |
|---|---|
| 空值 | 占位提示，不报错 |
| 输入中 | 实时格式化，必要时轻提示 |
| 超出范围 | 输入框下方红色提示 |
| 余额不足 | 显示余额与充值 / 更换支付方式入口 |
| 协议未勾选 | 按钮 disabled，并提示需阅读协议 |

## 8. 开关组件

自动跟车、自动调仓、提醒开关等高风险状态变更，不建议单击立即生效。

建议：

```text
[开关]
  ↓
[底部弹层说明影响]
  ↓
[确认开启 / 确认关闭]
```

## 9. 禁用

- 不隐藏输入限制；
- 不让用户在不知道费用、时间、风险的情况下提交；
- 不用 placeholder 代替字段标题；
- 不在金融表单里使用过强营销话术。
