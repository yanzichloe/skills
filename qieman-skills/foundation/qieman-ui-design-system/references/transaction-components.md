# Transaction Components 交易业务组件

## 1. 定义

交易业务组件用于买入、卖出、转换、调仓、定投、智能买入、智能回购、自动跟车等高风险金融操作。

## 2. 核心组件

| 组件 | 用途 |
|---|---|
| Strategy Header | 策略名称、服务方、风险等级 |
| Account Selector | 账户、银行卡、盈米宝、养老金账户 |
| Amount Input | 金额输入与快捷金额 |
| Fee Preview | 预估手续费、投顾服务费 |
| Date Preview | 预计确认 / 到账 / 完成日期 |
| Risk Notice | 风险提示、适当性提示 |
| Condition Card | 跌幅买入、回购条件 |
| Agreement Checkbox | 协议确认、授权确认 |
| Confirm Summary | 交易确认摘要 |
| Result Card | 提交成功 / 失败 / 处理中 |

## 3. 交易页面标准结构

```text
[顶部导航]
[策略 / 账户摘要]
[金额 / 条件输入]
[交易概览]
[费用 / 日期 / 支付方式]
[风险提示]
[协议确认]
[底部主按钮]
```

## 4. 金额组件

金额组件必须展示：

- 当前输入金额；
- 最小金额；
- 可用余额；
- 费用估算；
- 错误原因。

错误示例：

```text
余额不足，请更换支付方式或调整金额
买入金额需不低于 100 元
请输入 0-99% 的净值跌幅
```

## 5. 交易概览组件

交易概览应包含：

| 字段 | 说明 |
|---|---|
| 完成日期 | 预计完成日期及交易日说明 |
| 预估手续费 | 明确“预估”口径 |
| 到账方式 | 钱流路径与预计到账时间 |
| 支付方式 | 当前支付账户与余额状态 |

## 6. 智能买入 / 回购条件组件

条件组件结构：

```text
[条件名称]
[触发条件]
[触发后动作]
[有效期]
[金额]
[说明]
```

示例：

```text
净值较 2026-05-30 净值(1.9046)累计跌幅 ≥10%(1.7141) 时触发，并于下一交易日自动买入本策略。
```

## 7. 风险提示规范

交易组件必须明确：

- 投顾服务不承诺收益；
- 市场波动风险；
- 交易结果以基金公司确认为准；
- 费用为预估时必须说明；
- 自动计划需说明有效期与触发后动作。

## 8. Token Mapping

| 类型 | Token |
|---|---|
| 主操作 | `background/primary` |
| 风险文字 | `foreground/error` |
| 风险背景 | `background/error faded` |
| 警告文字 | `foreground/warning` |
| 警告背景 | `background/warning faded` |
| 成功状态 | `foreground/success` |
| 金额文字 | `foreground/neutral` |
| 辅助说明 | `foreground/neutral subtler` |

## 9. 禁用

- 不把“预估收益”作为主按钮诱导；
- 不隐藏费用和到账时间；
- 不使用“稳赚”“低风险高收益”等表达；
- 不让交易按钮在协议未勾选时可点击；
- 不让关闭自动计划等高风险操作单击即生效。
