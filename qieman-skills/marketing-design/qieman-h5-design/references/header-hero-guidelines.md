# 且慢营销 H5 首屏 Hero 区规范

## Purpose
本文件用于定义且慢营销 H5 页面首屏 Hero 区的设计规则，帮助在品牌传播页、专题页、活动页、年度账单页等场景中，统一首屏的信息顺序、标题层级、视觉焦点、CTA 位置、主视觉素材使用方式与氛围表达。

首屏 Hero 区通常决定用户是否愿意继续下滑，因此需要同时满足以下目标：

- 一眼理解页面主题
- 快速建立品牌感
- 提供继续阅读或操作的动机
- 保持视觉清晰与节奏感
- 在首屏优先使用本地 assets 中可复用且风格一致的主视觉素材

---

## When to read
在以下情况应优先读取本文件：

- 需要设计营销 H5 首页首屏
- 需要梳理标题、摘要、CTA 和主视觉之间的关系
- 需要设计人物、插画、IP、图标或氛围图的首屏构图
- 需要提升首屏转化效率和视觉吸引力
- 需要把 H5 文案转为首屏视觉结构
- 需要在 HTML Demo / 页面原型中确定首屏主视觉素材的调用方式

## Key rules

### 1. 首屏必须先讲主题，再讲细节
用户进入页面后，应在极短时间内理解：

- 这是什么页面
- 为什么值得看
- 下一步可以做什么

### 2. 首屏信息不宜过满
首屏重点不是「把所有内容都放进去」，而是建立价值感和阅读预期。

### 3. 标题层级必须明确
首屏通常由以下信息层级构成：

1. 标签 / 年份 / 场景标识
2. 主标题
3. 副标题 / 一句话说明
4. CTA
5. 主视觉（图标 / 人物 / 插画 / 氛围图 / 数据意向图）

### 4. 主视觉服务于标题，而不是与标题竞争
无论是人物、IP、插画、图标还是氛围图，都应该增强首屏主题，而不是抢占阅读焦点。

### 5. Hero 区必须保留留白
首屏必须保留足够的呼吸感，让标题、按钮和主视觉彼此分离且关系清楚。

### 6. 首屏必须使用本地 assets（强制）
- 首屏主视觉**仅允许**来自 `assets/`：先搜索 `assets/` 再落稿；优先用户提供的**精确路径**；不得忽略素材库；使用**实际存在的**文件与路径；禁止泛化替代。
- logo / IP / 插画 / 图标 / 背景 / UI / 图表**仅**从 assets 选用，保持且慢品牌一致；若匹配素材存在则**必须使用**。
- 若某类素材在 `assets/` 中**不存在**，必须在输出中**显式写出**「缺失素材：［类型/用途］」，不得静默用其他元素顶替。

## Hero asset rules

### 1. 首屏主视觉优先级（必须遵守）
当需要在 Hero 区放置视觉元素时：

1. **先搜索** `assets/`（及 `assets/library/`），再决定使用哪一档。
2. 用户已指定素材路径时，必须使用该**精确路径**。
3. 选用顺序：用户指定路径 → `assets/library/icons/` 匹配图标 → `assets/library/characters/` 人物/IP → `assets/library/scenes/` 场景图 → `assets/library/decorations/` 装饰。
4. 若库内**无**匹配素材：在方案或代码中**显式写出**「缺失素材：［类型/用途］」，不得用 emoji、占位图或通用图替代。

### 2. 图标型主视觉适用场景
优先用于以下页面：

- 年度账单页
- 数据总结页
- 收益概览页
- 配置总览页
- AI / 智能投顾专题页
- 金融信息导向型活动页

图标型主视觉特点：

- 更克制
- 更适合金融科技感
- 适合与大标题并列
- 不会过度占据情绪空间

### 3. 人物 / IP 型主视觉适用场景
优先用于以下页面：

- 品牌活动页
- 情绪型传播页
- 节日营销页
- 以「小顾」陪伴感为核心的专题页

要求：

- 不抢标题
- 动作和表情服务于主题
- 不做过度夸张表演
- 保持且慢品牌的温和、专业和陪伴感

### 4. 场景型主视觉适用场景
优先用于以下页面：

- 节日专题
- 品牌故事页
- 强氛围营销页
- 大主题活动页

要求：

- 用于增强环境氛围
- 不能压住标题与 CTA
- 场景色需与品牌蓝体系协调

### 5. 装饰元素的使用边界
装饰元素只用于辅助，不应成为首屏主体。例如：

- 光感渐变
- 柔和圆点
- 节庆纹样
- 浅层浮动几何图形

不要用过多装饰破坏金融品牌秩序感。

### 6. HTML Demo 中的素材调用要求（强制）
在 HTML Demo / 页面原型中：

- **仅使用**本地 `assets/` 下的**精确文件路径**（如 `../assets/library/icons/cool/icon-cool-bar-chart-growth-v01.png`），禁止占位 URL、泛化名称或通用破图。
- 首屏主视觉、图标、人物、装饰**仅**从 `assets/library/` 选取；若库内无合适项，在注释或说明中**显式写出**「缺失素材：［具体说明］」。
- 不要使用 emoji 充当正式主视觉；不要使用浏览器默认破图。
- HTML 在 `output/`、素材在 `assets/` 时，使用正确相对路径（如 `../assets/...`）；若预览需稳定路径，可将首屏所需素材复制到 `output/` 同级后引用。

## Recommended structure

### 标准首屏结构
```text
标签 / 年份
主标题
副标题 / 一句话说明
CTA 按钮
主视觉（图标 / 人物 / 插画 / 氛围图 / 数据图形）
```

布局建议：左文右图，或左图右文；文字与主视觉顶对齐（`align-items: flex-start`）；左侧留白 ≥ 40% 画布宽度。

### 年度账单类首屏推荐结构
```text
主标题（如「你的 2025 投资账单」）
副标题（如「一年投资，一份总结」）
截至日期 / 统计口径（小字）
主 CTA（如「查看我的账单」）
主视觉：数据 / 增长意向图标（来自 assets/library/icons/cool/）
```

背景建议：浅色渐变（A100–A200 品牌蓝）；主标题 26pt 加粗；副标题 16pt 常规；主视觉置于右上象限，比例克制，不抢标题。

### 活动专题类首屏推荐结构
```text
活动标签 / 节日标识（可选）
主标题（活动名称 / 利益点）
副标题（一句话说明 / 时间范围）
主 CTA（如「立即参与」）
主视觉：人物 / 小顾 IP 或场景图（来自 assets/library/characters/ 或 scenes/）
```

背景建议：根据活动调性选用品牌蓝、科技紫或节庆色板；人物类主视觉置于右侧中右区，与主标题基线对齐。

### AI / 数据型专题推荐结构
```text
功能标签（如「智能投顾」）
主标题（功能名 / 价值点）
副标题（一句话说明）
主 CTA（如「立即体验」）
主视觉：AI / 数据意向图标（来自 assets/library/icons/cool/）
```

背景建议：浅紫或浅蓝渐变（科技感）；图标型主视觉置于右上象限，宽高约 20%～26% 画布，顶与主标题齐平或略高。

## Layout guidelines

- **文字与主视觉顶对齐**：左侧文字块与右侧主视觉在垂直方向上顶对齐（`align-items: flex-start`），避免中线或底对齐导致的错位感。
- **左侧留白**：左侧及中部保留不少于 40% 画布宽度的空间给主题文字，主视觉不侵入文字区。
- **分区约定**：左侧为主题文字区（主标题、副标题、关键信息）；右上角或右侧为主视觉区（图标 / 人物 / 插画）。
- **背景**：浅色系为主，确保标题与副标题易读、醒目；深色 Hero 时须使用白字并保证对比度。
- **构图原则**：文字为主、主视觉为辅，信息层级清晰；关键区域比例约 4:3（文 : 图）。

## Title hierarchy

| 层级         | 字号   | 字重   | 颜色（浅色底） | 颜色（深色底） |
|--------------|--------|--------|----------------|----------------|
| 主标题       | 26pt   | 700    | #333333        | #FFFFFF        |
| 副标题       | 16pt   | 400    | #606060        | rgba(255,255,255,0.9) |
| 标签/日期    | 14pt   | 400    | #999999        | rgba(255,255,255,0.7)  |

主标题可自适应换行，不设 `white-space: nowrap`；主副标题间距 8px，副标题与备注 4px。窄屏时主标题可用 `clamp(18px, 5.2vw, 26px)` 做字号适配。

---

## CTA guidelines

- **主 CTA**：背景 `#1B88EE`，文字 `#FFFFFF`，圆角 8px 或胶囊形 `border-radius: 9999px`，内边距 12px 24px；无边框、无投影。
- **次 CTA**：文字 `#1B88EE` 或轻底 `rgba(27,136,238,0.1)`，边框可选 `1px solid #9CCBF8`。
- **一屏一主 CTA**：首屏最多一个主按钮，避免多个强操作竞争注意力。
- **位置**：CTA 通常位于副标题下方、主视觉左侧或下方，与文字区对齐。

---

## Hero visual size rules

| 类型       | 位置           | 比例（占画布）           | 高度锚点说明 |
|------------|----------------|--------------------------|--------------|
| 图标类     | 右上象限       | 宽约 20%～26%，高约 20%～26% | 顶与主标题齐平或略高，底在画面水平中线之上 |
| 人物类     | 右侧中右区     | 高约 50%～55%（头至腰），宽约 30%～45% | 主标题基线与人物眼睛/额头同高，躯干延伸至中段 |

图标偏紧凑、偏上；人物偏饱满、偏下；二者在视觉锚点与占比上需明显区分，不可混用同一套比例。

---

## Brand style requirements

- **基调**：专业、可信、轻盈、现代、有秩序感；营销感不能破坏金融品牌可信度。
- **色彩**：与《且慢营销配色规范》一致；主 CTA 固定品牌蓝 `#1B88EE`；背景使用 100–200 档浅色或深色 Hero 渐变。
- **禁止**：高饱和撞色、霓虹渐变、厚重投影、强情绪化或戏谑夸张的人物表情、主视觉压住标题与 CTA。
- **素材风格**：与且慢 AI 3D 素材 / 品牌视觉同源；同一首屏内图标或人物风格统一。

---

## Dos

- **必须**使用本地 `assets/` 下的主视觉素材（先搜 assets，用精确路径；无则写「缺失素材：…」）
- 左文右图顶对齐，左侧留白充足
- 主标题、副标题、CTA、主视觉层级清晰
- 首屏一句话讲清主题与价值
- 深色 Hero 时保证文字对比度
- HTML Demo 中首屏图片使用 `assets/` 下实际存在的路径（必要时复制到 output 同级以保证可解析）

---

## Don'ts

- 不要用 emoji、占位图或通用图充当正式首屏主视觉（仅允许 assets 内素材）
- 不要主视觉与标题抢焦点或压字
- 不要首屏堆满信息、缺少留白
- 不要使用与品牌风格不一致的外部占位图或默认破图
- 不要忽略素材库：有匹配素材则必须使用；无则显式写「缺失素材」
- 不要在同一首屏混用多套主视觉比例（如图标与人物按同一尺寸摆放）
- 不要忽略数据口径或截至时间（年度账单、数据类专题应标注）

## Examples

### 示例 1：年度账单 Hero
- **主标题**：你的 2025 投资账单  
- **副标题**：一年投资，一份总结  
- **备注**：截至 2025-12-31（14pt #999999）  
- **主 CTA**：查看我的账单（品牌蓝胶囊按钮）  
- **主视觉**：`assets/library/icons/cool/icon-cool-bar-chart-growth-v01.png`，置于右上角，约 100×100px（或画布宽 20%～26%），与主标题顶对齐  
- **背景**：线性渐变 `#F5F9FF → #DBEBFF → #F5F9FF`，165deg  
- **布局**：左文右图，文字区 ≥40% 宽度，主视觉不侵入左侧

### 示例 2：活动专题 Hero
- **主标题**：新春理财季  
- **副标题**：开年好礼，稳健启程  
- **主 CTA**：立即参与（品牌蓝胶囊按钮）  
- **主视觉**：人物或小顾 IP（来自 `assets/library/characters/`），置于右侧中右区，人物高约画布 50%～55%，主标题基线与人物眼睛齐平  
- **背景**：浅色渐变（品牌蓝 A100–A200）或节庆红金 100–200；若用人物，背景色按人物主色适配（people / people-warm 等）  
- **布局**：左文右图顶对齐，左侧留白充足

### 示例 3：AI 专题 Hero
- **主标题**：且慢智能投顾  
- **副标题**：数据驱动，理性配置  
- **主 CTA**：立即体验（品牌蓝胶囊按钮）  
- **主视觉**：AI / 数据意向图标（来自 `assets/library/icons/cool/`，如 ai-cube、仪表盘等），置于右上象限，宽高约画布 20%～26%，顶与主标题齐平或略高  
- **背景**：科技紫浅色渐变 `#F5F0FF → #E7DBFF` 或品牌蓝 A100–A200  
- **布局**：左文右图顶对齐，主视觉克制不抢标题
## Enforcement

- 生成首屏方案前，必须先扫描 `assets/`
- 若找到匹配素材，必须优先使用，不得替换为生成元素
- 输出时必须显式写出所用素材的精确路径
- 禁止使用远程图片、base64、placeholder、默认破图
- 若未找到匹配素材，必须输出：`缺失素材：［类型/用途］`
### Local preview fallback
当 HTML Demo 位于 `output/` 且本地预览需要稳定展示时，可将首屏所需素材复制到 `output/` 同级目录，并优先使用同级相对路径引用，如 `./icon-xxx.png`，以避免本地预览环境下跨目录引用失效。
## Asset usage boundary

- `assets/` 下素材可作为页面视觉资源池统一调用，但不得脱离当前页面场景、信息层级、品牌风格与模块用途随意使用。
- 素材调用应先匹配页面类型，再匹配素材类别与风格一致性。
- 不得因素材存在而强行使用；若不适配当前页面，应跳过并继续检索更合适素材。
- 同一页面内应保持素材风格统一，不混用明显不同风格的图标、人物、场景与装饰素材。
- 首屏 Hero 优先选择与页面主题最匹配的主视觉素材，正文模块再按信息层级使用辅助素材。
- 节日、活动、情绪化素材不得直接复用于通用金融说明页、数据总结页或理性配置页，除非页面主题明确匹配。
- 装饰素材只能作为辅助视觉元素，不得替代主视觉、核心图标或关键信息表达。
- 若 `assets/` 中存在多个可用素材，应优先选择品牌一致性最高、语义匹配最强、构图最适合当前模块的一项。
