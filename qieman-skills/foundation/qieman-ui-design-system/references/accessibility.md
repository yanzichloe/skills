# Accessibility 无障碍设计规范

## 1. 目标

且慢组件系统必须保证不同年龄、视觉能力、操作习惯的用户都能理解和完成关键金融操作。

## 2. 可读性

| 项目 | 规则 |
|---|---|
| 正文字号 | App 正文建议 14px - 16px |
| 辅助说明 | 不低于 12px |
| 行高 | 正文建议 1.45 - 1.6 倍 |
| 金额数字 | 需要足够大，避免误读 |
| 风险提示 | 不得使用过低对比度 |

## 3. 触控区域

- 可点击区域不小于 44×44px；
- 关键按钮高度建议 48px；
- 多个操作并列时保证间距；
- 勾选框、开关、帮助图标不可过小。

## 4. 对比度

- 普通正文与背景对比应满足基本可读性；
- 风险、警告、成功不只依赖颜色，还要配合文字或图标；
- 浅色背景上的辅助文字不可过浅。

## 5. 状态可感知

所有交互组件必须提供：

```text
默认态 / 按下态 / 禁用态 / 加载态 / 错误态 / 成功态
```

屏幕阅读器语义建议：

- Button 使用明确动作名称；
- Input 提供 label；
- Error 与对应输入框绑定；
- Toast / Alert 使用可识别的状态语义。

## 6. 金融信息可理解

金融组件必须减少认知负担：

- 不使用过多专业术语；
- 必要术语提供解释入口；
- 费用、到账时间、交易状态必须可见；
- 风险提示不应只放在页面底部小字中。

## 7. AI 可解释

AI 组件需说明：

- 结论来自哪些数据；
- 是否可能不准确；
- 用户可以如何验证；
- 是否需要咨询顾问或重新测评。

## 8. 禁用

- 不用纯颜色表达涨跌 / 成败；
- 不让重要说明只在 hover 中出现；
- 不在移动端使用过小点击图标；
- 不隐藏焦点状态；
- 不用晦涩系统错误码直接面向用户。
