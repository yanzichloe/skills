# Color System / 色彩系统规范

## 1. 色彩原则

且慢色彩系统应表达：

```text
专业
可信
清爽
克制
长期陪伴
```

主色以品牌蓝为核心，结合白色、浅灰、浅蓝背景建立轻金融感。

## 2. 主色

| Token | 色值 | 用途 |
|---|---:|---|
| brand-primary | `#1B88EE` | 主按钮、核心 CTA、品牌识别、重点信息 |

## 3. 基础色

| Token | 色值 | 用途 |
|---|---:|---|
| bg-page | `#F9FAFB` | App / H5 / Web 页面底色 |
| bg-card | `#FFFFFF` | 卡片、弹窗、内容容器 |
| text-title | `#333333` | 一级标题、重点文本 |
| text-body | `#606060` | 正文、普通说明 |
| text-secondary | `#999999` | 辅助说明、备注、时间 |
| text-disabled | `#CCCCCC` | 禁用、弱提示 |
| border-divider | `#F0F2F5` | 分割线、卡片边界 |

## 4. 状态色

| Token | 色值 | 用途 |
|---|---:|---|
| state-error | `#FA440C` | 错误、失败、风险警示 |
| state-warning | `#EA9500` | 注意事项、重要提醒 |
| state-success | `#07AD8F` | 成功反馈、确认状态 |

## 5. 辅助色比例

| 色系 | 建议比例 | 使用场景 |
|---|---:|---|
| 浅蓝 | 60% - 80% | 背景、模块底色、品牌氛围 |
| 白色 | 20% - 40% | 信息承载、卡片、留白 |
| 暖黄 / 浅橙 | ≤ 20% | 温度感、权益、节日、轻营销 |
| 浅紫 | ≤ 15% | AI、科技、情绪化表达 |
| 金色 | 小面积点缀 | 高净值、徽章、荣誉、权益感 |

## 6. 推荐渐变

```css
--gradient-blue: linear-gradient(180deg, #C7E3FF 0%, #F0F6FF 100%);
--gradient-warm: linear-gradient(180deg, #FFF1D4 0%, #FEF4FC 100%);
--gradient-purple: linear-gradient(180deg, #DFE1FF 0%, #F1F2FB 100%);
--gradient-orange: linear-gradient(180deg, #FFE5D2 0%, #FFF6F0 100%);
```

## 7. 使用规则

### 7.1 主按钮

- 使用 `#1B88EE`；
- 不建议使用红色、绿色、金色作为主 CTA；
- 禁用态可使用浅灰或低透明度主色。

### 7.2 金融涨跌与风险

- 风险、错误、失败使用 `#FA440C`；
- 重要提醒使用 `#EA9500`；
- 成功反馈使用 `#07AD8F`；
- 不建议大面积使用绿色，避免与金融涨跌认知、交易暗示发生冲突。

### 7.3 背景

- 优先使用 `#F9FAFB`；
- 首屏营销可使用浅蓝渐变；
- 高净值可使用深蓝、蓝金，但需克制；
- 避免大面积纯黑和强金。

## 8. 禁用倾向

- 禁止纯黑作为大面积背景或文字主色；
- 避免高饱和红绿对比；
- 避免强金、强紫、强红造成低质营销感；
- 避免用色彩暗示“稳赚”“暴富”“确定性机会”。
