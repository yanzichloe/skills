# Visual Language / 视觉语言规范

## 1. 视觉核心

且慢视觉语言围绕以下关键词建立：

```text
轻金融
可信任
陪伴感
长期感
低压感
数据可解释
```

## 2. 推荐视觉关键词

- 轻量化
- 简洁主义
- 柔和渐变
- 大面积留白
- 圆角卡片
- 柔光质感
- 低压金融感
- 数据可解释
- 自然陪伴感

## 3. 页面结构原则

### 3.1 信息层级

优先让用户快速理解：

1. 当前状态是什么；
2. 和我有什么关系；
3. 风险或影响是什么；
4. 下一步可以怎么做。

### 3.2 卡片化表达

推荐使用白色或浅蓝卡片承载信息：

- 卡片背景：`#FFFFFF` / `#F9FAFB` / 浅蓝渐变；
- 圆角：柔和，不尖锐；
- 阴影：轻，不制造厚重感；
- 边界：弱描边或无描边。

### 3.3 留白

留白是且慢品牌重要特征。

- 页面不要被信息塞满；
- 金融数据要分组展示；
- 关键 CTA 周围保留足够呼吸感；
- 海报与 H5 首屏建议保留 50% 以上视觉呼吸空间。

## 4. 图表语言

金融图表应以“帮助理解”为目的。

推荐：

- 坐标轴清晰；
- 关键数据突出；
- 收益与风险并列；
- 口径说明可见；
- 颜色克制，避免过度炫彩。

避免：

- 只突出收益，不展示风险；
- 用视觉夸大收益曲线；
- 强刺激红绿对比；
- 装饰元素干扰读数。

## 5. 组件质感

| 组件 | 视觉要求 |
|---|---|
| 按钮 | 主按钮统一品牌蓝，层级明确 |
| 弹窗 | 克制、清晰、帮助理解，不强推 |
| 卡片 | 圆角、轻阴影、浅背景 |
| 列表 | 间距清晰，信息扫描效率高 |
| 输入框 | 简洁、稳定、状态明确 |
| 标签 | 颜色轻，不做强刺激 |

## 6. 跨端一致性

App、H5、Web、海报、报告应保持统一品牌识别：

- 主色一致；
- 字体气质一致；
- 圆角与卡片质感一致；
- 关键图形语言一致；
- 风险提示表达一致；
- AI / 投顾 / 交易等重点业务的语气一致。
