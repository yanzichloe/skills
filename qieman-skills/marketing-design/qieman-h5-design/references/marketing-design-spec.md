# 且慢营销 H5 设计通用规范

## Purpose
本文件用于定义且慢营销 H5 页面在整体结构、模块节奏、信息优先级、转化路径与品牌一致性上的通用规则，帮助不同主题、不同活动、不同周期的 H5 页面保持统一的品牌体验与设计质量。

本规范重点解决以下问题：

- H5 长页应该如何组织信息
- 页面模块如何分段与衔接
- 用户阅读路径如何建立
- 营销表达与品牌可信度如何平衡
- 页面如何兼顾转化目标与视觉体验

## When to read
在以下情况应优先读取本文件：

- 需要从 0 到 1 规划营销 H5 页面
- 需要整理页面结构、模块顺序、信息优先级
- 需要将一份营销文案转成页面原型
- 需要统一不同营销页面的设计质量
- 需要判断页面中过多或不足的模块安排
- 需要为 HTML Demo / 页面原型确定整体骨架与模块顺序

## Key rules

### 1. 页面必须有明确目标
每个 H5 页面都应先明确其核心目标，例如：

- 传播品牌理念
- 介绍产品 / 服务
- 激发领取 / 点击 / 跳转
- 展示年度回顾
- 强化用户参与感

### 2. 页面结构服务于目标，不服务于堆料
不要把所有信息都平铺进页面。页面应只保留对目标有帮助的内容。

### 3. 信息顺序应遵循「先价值，后细节」
推荐顺序：

1. 页面主题
2. 价值表达
3. 证据 / 数据 / 说明
4. 深入内容
5. 转化动作
6. 风险提示 / 免责说明

### 4. 长页必须有节奏
长页中应通过模块切换、卡片节奏、留白和视觉变化形成阅读推进，而不是一屏屏重复堆砌。

### 5. 营销表达不能削弱金融品牌专业性
即使在节日、活动、热点场景中，也应保留且慢品牌应有的：

- 专业
- 可信
- 克制
- 清晰
- 有秩序感

## Recommended structure

### 标准营销 H5 页面结构
1. 首屏 Hero 区  
2. 价值亮点区  
3. 数据 / 证据区  
4. 详细说明区  
5. 转化引导区  
6. 底部说明区  

首屏建立主题与价值感；价值亮点与数据区强化信任；详细说明按需展开；转化引导区集中 CTA；底部说明区含免责与数据口径。

### 年度账单类页面推荐结构
1. 首屏 Hero（主标题 + 副标题 + 截至日期 + 主 CTA + 数据意向图标）  
2. 年度亮点摘要卡（累计投入、收益率、资产峰值、投资天数等）  
3. 资产配置 / 数据可视化区（环形图或柱状图 + 图例 + 数据来源）  
4. 年度时间线或关键节点  
5. 用户投资画像 / 关键词 / 对比（可选）  
6. 结尾 CTA（生成海报、分享账单等）  
7. 底部免责与数据口径  

适合移动端长页滚动；模块间距 16px；数据类模块必附统计周期与数据来源。

### 活动专题类页面推荐结构
1. 首屏 Hero（活动主题 + 副标题 + 主 CTA + 人物/场景主视觉）  
2. 活动卖点卡 ×2～3  
3. 流程或规则说明  
4. 权益清单或对比（可选）  
5. 主 CTA 区  
6. 风险提示 / 免责  

活动页可适度增强氛围，但主 CTA 仍为品牌蓝，不改变按钮规范。

## Module rhythm
- 模块之间保持统一间距（建议 16px），避免忽密忽疏。
- 同一层级的信息块（如多张卡片）视觉重量尽量一致，避免单块过于突出或弱化。
- 通过分区标题（蓝竖条 + 19pt）、顶条标题卡、留白分隔形成节奏，让用户有「段落感」。
- 长页中可穿插 2～3 个强转化节点（如首屏 CTA、中部 CTA、底部 CTA），但一屏内主 CTA 不超过一个。
- 底部说明区（免责、数据口径）用小字号、次级色，与主体内容明显区分。

## Information priority
1. **最高**：首屏主标题、核心价值句、主 CTA。  
2. **高**：关键数据、卖点标题、分区标题。  
3. **中**：正文说明、图例、列表项。  
4. **低**：数据来源、统计周期、辅助说明。  
5. **最低**：免责声明、风险提示、页脚说明。  

同一模块内：标题 > 核心数值/卖点 > 补充说明 > CTA/辅助。确保用户能按优先级扫读。

## Conversion path
- 首屏即给出主行动（如「查看我的账单」「立即参与」），减少犹豫。
- 长页中在关键结论或权益说明后，适时重复 CTA（如「生成年度海报」「分享账单」），但形式可区分主次按钮。
- 主 CTA 使用品牌蓝 `#1B88EE`，胶囊形或 8px 圆角；次 CTA 使用轻底或描边，不抢主按钮。
- 转化路径应清晰：看到主题 → 理解价值 → 看到证据/数据 → 产生行动意愿 → 点击 CTA。避免在转化前堆砌过多无关信息。

## Brand consistency
- 色彩、字体、圆角、间距与《且慢营销配色规范》《卡片样式规范》《首屏 Hero 规范》保持一致。
- 主 CTA 固定品牌蓝，不随页面基调改变。
- 视觉基调：专业可信、克制清爽、数据友好、温和不压迫。
- 禁止：高饱和撞色、霓虹渐变、厚重投影、强情绪化拟人、误导性促销表达。
- 多页面、多活动共用同一套组件与版式逻辑，便于沉淀与复用。

## Dos
- 先定页面目标，再拆模块与信息优先级。
- 长页用分区标题与卡片节奏形成阅读节奏。
- 数据类内容必附数据来源与统计周期。
- 主 CTA 明确、位置合理，一屏一主按钮。
- 底部保留免责与风险提示，小字号可见。
- 与 header-hero、card-styles、color-guidelines 等 reference 配合使用，保持整体一致。

## Don'ts
- 不要无目标地堆砌模块或信息。
- 不要忽略移动端长页的节奏感，避免整页同质化。
- 不要省略数据口径或免责说明。
- 不要在同一屏放置多个主 CTA 竞争注意力。
- 不要用强促销、强情绪化表达削弱金融品牌可信度。

## Examples

**示例 1：年度账单 H5 整体结构**  
Hero（你的 2025 投资账单 + 查看我的账单 + 数据图标）→ 年度亮点卡（四格数据）→ 资产配置卡（环形图 + 图例 + 数据来源）→ 年度时间线卡 → 投资画像卡（关键词 + 柱状图 + 数据来源）→ CTA 区（生成年度海报 + 分享账单）→ 底部免责。模块间距 16px，数据区均带统计周期。

**示例 2：活动专题 H5 整体结构**  
Hero（活动名称 + 副标题 + 立即参与 + 人物/场景图）→ 卖点卡 ×3（图标 + 标题 + 要点）→ 流程或规则卡 → 权益清单（可选）→ 主 CTA 区 → 风险提示。主按钮统一品牌蓝，活动氛围用背景与插图体现，不改变 CTA 规范。

**示例 3：功能宣发 H5 整体结构**  
Hero（功能名 + 一句话价值 + 立即体验 + AI/数据图标）→ 卖点卡 ×3 → 权益或对比卡 → FAQ（可选）→ CTA → 风险提示。信息优先级：主题 > 价值 > 证据 > 转化 > 免责。
